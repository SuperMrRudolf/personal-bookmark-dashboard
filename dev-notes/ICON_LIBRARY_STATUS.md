# Icon Library Status

## 1. Why We Save Icons Locally

I save icons locally in the extension because normal favicon lookup does not always return the correct product icon.

For example:
- a Google Drive bookmark can end up showing the generic Google icon
- a Google Sheets bookmark can end up showing the generic Google icon
- services that live under the same main domain often fall back to the parent site's favicon instead of the specific app icon

That was the original reason for introducing local icon mapping: I want known services to show the correct app icon, not just whatever favicon the main domain returns.

I also save icons locally so bookmark icons load instantly, work more reliably, and do not depend on network requests every time a new tab opens.

Benefits:
- correct app-specific icons for known platforms
- faster icon loading
- fewer broken icons from blocked or changed favicon URLs
- more consistent visuals for known platforms
- better offline behavior
